Heap Sort


Accessibility Settings







Description

Heap sorts are sorting algorithms that utilize binary trees to sort the array. It creates a max binary tree, then removes the max element to place it last in the resulting array. Its time complexity is O(N log N).

Heap sorts are a comparison-based sorting algorithm, where it divides its sorted portion between its unsorted portions, by pulling the largest elements from the unsorted portion into the sorted list.

It follows a comparison-based sorting technique based off of binary heap data structures. Similarities coincide with selection sort where the first minimum elements will be placed at the beginning of the sorted lists. Heapsorts are unstable sorting algorithms, due to the operations being able to change the order of equivalent keys in an array. This array-based heap can be used to help reduce storage and memory by overwriting inputs using no extra data structures at runtime.

Heapify: is the process of creating heap data structures from binary trees using arrays to create min-heap and max-heap array based structures. This is primarily used in implemenation of the code to keep track of its largest function seperately from the sorting function of the algorithm. Heapsorts are built in a “Parent-Child” architecture in which the uppermost nodes are called the “parents” and their roots are labeled as their children. The start process of a heapsort is to create itself into a “max heap”, where the value of parent nodes is greater than or equal to the values of its children.

Functionality: Heap Sorts worst case performance is O(n log n), with its best being O(n log n) OR O(n). The difference between the O(n log n) and O(n) is the difference in keys the heap sort would need to parse through each iteration.

Advantages: Efficiency increases logarithmically while other algorithms grow slower as # of items to sort increases in complexity. Simpler to implement, and no additional memory usage is needed to input information.


Step-By-Step

1. Build a binary tree
2. Create a max heap
3. Remove the max element and place at the end of the resulting array
4. Repeat steps 2 and 3 until all elements have been removed
